Progress indicators

Adding progress indicators to sections to inform the user when their data will be saved

Progress indicators prototype

We're conducting this research based on feedback from the A/B User Journey (task list) research. The users prefered the question-by-question approach of filling in the forms, but they were not aware their questions weren't saved until the end of each section of questions.

Although the ideal journey would be to save each question and to add an 'in progress' indicator to the task list, we're restricted by TPR's database (i.e. Registry) which is not able to save in this way.

During the A/B User Journey (task list) research, we had feedback from users that progress indication within these sections would help. We've decided to conduct more research in this area so that what we deliver is as close to the preferred user journey as possible within our limitations.

What we're looking for

  1. Is it obvious when your work will save?
  2. Does the progress indicator help?
